API

To use the functionality of DimensionalData in your module, dispatch on AbstractDimensionalArray and AbstractDimension.

DimensionalArray(data, dims, refdims, name)

The main subtype of AbstractDimensionalArray. Maintains and updates its dimensions through transformations and moves dimensions to refdims after reducing operations (like e.g. mean).

Getting basic info

These useful functions for obtaining information from your dimensional data:

DimensionalData.dims — Function
dims(x)

Return a tuple of the dimensions for a dataset. These can contain the coordinate ranges if bounds() and select() are to be used, or you want them to be shown on plots in place of the array indices.

They can also contain a units string or unitful unit to use and plot dimension units.

This is the only method required for this package to work. It probably requires defining a dims field on your object to store dims in.

DimensionalData.refdims — Function
refdims(x)

Reference dimensions for an array that is a slice or view of another array with more dimensions.

slicedims(a, dims) returns a tuple containing the current new dimensions and the new reference dimensions. Refdims can be stored in a field or disgarded, as it is mostly to give context to plots. Ignoring refdims will simply leave some captions empty.

Selectors

DimensionalData.Selector — Type

Selectors indicate that index values are not indices, but points to be selected from the dimension values, such as DateTime objects on a Time dimension.

source
DimensionalData.At — Type
At(x)

Selector that exactly matches the value on the passed-in dimensions, or throws an error. For ranges and arrays, every value must match an existing value - not just the end points.

source
DimensionalData.Near — Type
Near(x)

Selector that selects the nearest index. With PointSampling this is simply the index nearest to the contained value, however with `IntervalSampling it is the interval center nearest to the contained value. This will be offset from the index value for Start and End loci.

source
DimensionalData.Between — Type
Between(a, b)

Selector that retreive all indices located between 2 values.

For IntervalSampling the whole interval must be lie between the values. For PointSampling the points must fall between the 2 values. These different sampling traits will often give different results with the same index and values - this is the intended behaviour.

source

Tracking the order of arrays and indices:

DimensionalData.Unordered — Type

Trait indicating that the array or dimension has no order.

source
DimensionalData.Ordered — Type

Trait container for dimension and array ordering.

The default is Ordered(Forward(), Forward(), Forward())

All combinations of forward and reverse order for data and indices seem to occurr in real datasets, as strange as that seems. We cover these possibilities by specifying the order of both explicitly.

Knowing the order of indices is important for using methods like searchsortedfirst() to find indices in sorted lists of values. Knowing the order of the data is then required to map to the actual indices. It's also used to plot the data later - which always happens in smallest to largest order.

Base also defines Forward and Reverse, but they seem overly complicated for our purposes.

DimensionalData.reducedims — Function

Replace the specified dimensions with an index of length 1 to match a new array size where the dimension has been reduced to a length of 1, but the number of dimensions has not changed.

Used in mean, reduce, etc.

Grid traits are also updated to correspond to the change in cell step, sampling type and order.

source
DimensionalData.slicedims — Function

Slice the dimensions to match the axis values of the new array

All methods returns a tuple conatining two tuples: the new dimensions, and the reference dimensions. The ref dimensions are no longer used in the new struct but are useful to give context to plots.

Called at the array level the returned tuple will also include the previous reference dims attached to the array.
